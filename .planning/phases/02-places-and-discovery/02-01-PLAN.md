**Place.swift — replace stub entirely:**
- Keep `Identifiable, Equatable` conformance, keep `id: UUID`
- Add fields: `name: String`, `coordinate: CLLocationCoordinate2D`, `phoneNumber: String?`, `websiteURL: URL?`, `address: String` (formatted from placemark components: subThoroughfare + thoroughfare + locality), `distanceMeters: Double`
- Add `init(from mapItem: MKMapItem, userLocation: CLLocation)` that maps all fields from the MKMapItem. Name falls back to "Unknown" if nil.
- Add computed property `distanceInPizzaSlices: Int` — `Int((distanceMeters / 0.2032).rounded())`. 1 pizza slice = 8 inches = 0.2032 meters.
- Add computed property `distanceDisplayString: String` — `"\(distanceInPizzaSlices.formatted()) slices away"`
- Keep `Equatable` via `lhs.id == rhs.id`
- Import MapKit
- Do NOT add rating or hours fields — MKMapItem does not expose them

**PlacesService.swift — replace stub entirely:**
- `@Observable @MainActor final class PlacesService`
- Published properties: `var places: [Place] = []`, `var isLoading = false`, `var error: PlacesError? = nil`
- Private state: `searchRegionRadiusKm: Double = 1.0`, `radiusStepKm: Double = 1.0`, `maxRadiusKm: Double = 10.0`
- Define `enum PlacesError: Error { case searchFailed(Error), noResults, noLocation }` at file scope
- Method `func fetchNearby(userLocation: CLLocation) async`:
  - Create `MKLocalSearch.Request()` with `naturalLanguageQuery = "pizza"`, `resultTypes = .pointOfInterest`, `pointOfInterestFilter = MKPointOfInterestFilter(including: [.restaurant])`, region from `userLocation.coordinate` with `searchRegionRadiusKm * 1000` meters
  - Execute `try await MKLocalSearch(request: request).start()`
  - Sort `response.mapItems` by distance from userLocation (ascending)
  - Map to `[Place]` via `Place(from:userLocation:)`
  - Set `places = sorted`. If empty, set `error = .noResults`
  - Wrap in do/catch — catch sets `error = .searchFailed(error)`
  - Set `isLoading = true` at start, `false` at end (both success and error paths)
  - Reset `searchRegionRadiusKm = 1.0` on each fresh fetch
- Method `func loadMore(userLocation: CLLocation) async`:
  - Guard `searchRegionRadiusKm < maxRadiusKm`, else return
  - Increment `searchRegionRadiusKm += radiusStepKm`
  - Execute same MKLocalSearch query with expanded region
  - Deduplicate against existing places by coordinate proximity (within 10 meters: compare `CLLocation.distance(from:) < 10`)
  - Append only new results, re-sort entire array by distance
- Method `func updateDistances(userLocation: CLLocation)`:
  - Rebuild `places` array with recalculated `distanceMeters` for each place WITHOUT re-querying MKLocalSearch
  - This means Place needs an internal init that copies all fields but updates distanceMeters. Add `func withUpdatedDistance(userLocation: CLLocation) -> Place` to Place.swift that creates a new Place with same id/name/coordinate/phone/url/address but recalculated distanceMeters
  - Re-sort places by new distance after updating
