  <action>
    Create PrivacyInfo.xcprivacy at TakeMeToPizza/Resources/PrivacyInfo.xcprivacy with this exact content:

    ```xml
    <?xml version="1.0" encoding="UTF-8"?>
    <!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN"
      "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
    <plist version="1.0">
    <dict>
        <key>NSPrivacyCollectedDataTypes</key>
        <array>
            <dict>
                <key>NSPrivacyCollectedDataType</key>
                <string>NSPrivacyCollectedDataTypePreciseLocation</string>
                <key>NSPrivacyCollectedDataTypeLinked</key>
                <false/>
                <key>NSPrivacyCollectedDataTypeTracking</key>
                <false/>
                <key>NSPrivacyCollectedDataTypePurposes</key>
                <array>
                    <string>NSPrivacyCollectedDataTypePurposeAppFunctionality</string>
                </array>
            </dict>
        </array>
        <key>NSPrivacyAccessedAPITypes</key>
        <array/>
        <key>NSPrivacyTracking</key>
        <false/>
    </dict>
    </plist>
    ```

    CRITICAL: The file must be added to the Xcode target's "Copy Bundle Resources" build phase. If editing project.pbxproj directly, add a PBXBuildFile reference for PrivacyInfo.xcprivacy in the Resources build phase. If using xcodebuild or xcodeproj tooling, ensure target membership is set.

    IMPORTANT distinction from research: Core Location goes in NSPrivacyCollectedDataTypes (data collected), NOT in NSPrivacyAccessedAPITypes (required reason APIs). Do not confuse these.
