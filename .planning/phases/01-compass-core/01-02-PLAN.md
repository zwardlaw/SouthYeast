  <name>Task 2: Create LocationService with permission state machine</name>
  <files>

  <action>
    Create LocationService.swift following the research patterns exactly. This is the most critical file in the app.

    Requirements:
    1. `@Observable @MainActor final class LocationService: NSObject`
    2. Published properties: `location: CLLocation?`, `heading: Double` (default 0.0), `headingAccuracy: Double` (default -1.0), `authorizationStatus: CLAuthorizationStatus` (default .notDetermined)
    3. Computed `permissionStatus: PermissionStatus` property mapping CLAuthorizationStatus to the app's PermissionStatus enum
    4. `PermissionStatus` enum: `.notDetermined`, `.denied`, `.restricted`, `.authorized` — define this in the same file or Place.swift
    5. Private `CLLocationManager` instance configured in init:
       - `desiredAccuracy = kCLLocationAccuracyNearestTenMeters`
       - `distanceFilter = 50.0` (meters)
       - `headingFilter = 1.0` (degrees)
       - `delegate = self`
    6. `startUpdating()`: Check authorization status first. If `.notDetermined`, call `requestWhenInUseAuthorization()`. If `.authorizedWhenInUse` or `.authorizedAlways`, call both `startUpdatingLocation()` AND `startUpdatingHeading()`. Both must run simultaneously for trueHeading to work.
    7. `stopUpdating()`: Stop both location and heading updates. Set `headingAccuracy = -1.0` to trigger calibration state on resume.

    CLLocationManagerDelegate extension (ALL methods must be `nonisolated`):
    1. `locationManagerDidChangeAuthorization`: Extract `manager.authorizationStatus` as a local, then `Task { @MainActor in }` to update `self.authorizationStatus`. Auto-start location+heading if status becomes authorized.
    2. `locationManager(_:didUpdateLocations:)`: Take `locations.last`, pass to main actor via Task. CLLocation is Sendable — safe to capture directly.
    3. `locationManager(_:didUpdateHeading:)`: Extract `newHeading.trueHeading` and `newHeading.headingAccuracy` as Doubles BEFORE the Task boundary (CLHeading is NOT Sendable). Inside Task: update `self.headingAccuracy`. Only update `self.heading` if `accuracy >= 0`.
    4. `locationManagerShouldDisplayHeadingCalibration`: Return `true`.
    5. `locationManager(_:didFailWithError:)`: Log but do not crash — location is optional.

    CRITICAL CORRECTNESS RULES (from research):
    - NEVER use `magneticHeading` — always `trueHeading`. `trueHeading` requires `startUpdatingLocation()` running simultaneously.
    - NEVER update `heading` when `headingAccuracy < 0` — this produces garbage compass directions.
    - CLHeading is NOT Sendable — extract scalar Doubles before Task boundary. CLLocation IS Sendable.
    - `headingAccuracy` defaults to -1.0 so the app starts in calibration state until the first valid heading arrives.
  </action>
